# 🧑‍💼 TeamSync – Human Resource Database System

> A modern HR Management System (Static Prototype) built using SRS, DFDs, and UML design principles.

---

## 🚀 Overview

**TeamSync** is a Human Resource Management System designed to handle the **complete employee lifecycle** in IT organizations.

It demonstrates **end-to-end system design**:
- 📊 Data Flow Diagrams (DFD)
- 🧩 UML Class Diagram
- 🔄 Sequence Diagram
- 🖥️ Functional UI Prototype

---

## ✨ Features

- 🔐 Authentication System (Admin / HR User)
- 📊 Dashboard with employee statistics
- ➕ Add Employee (Hiring Module)
- 🔍 Search Employee (Keyword + Filters)
- ✏️ Modify Employee Details
- 🚪 Employee Exit & Feedback Management
- 📁 Project Assignment System

## 🧠 System Design

### 📌 Modules

- **Authentication Module** → Login & session handling  
- **Hiring Module** → Add employee & biodata  
- **Search Module** → Search + assign to project  
- **Amendment Module** → Modify employee details  
- **Exit Module** → Exit processing & feedback  

---

### 🧩 Core Entities

- **User** – Authentication handling  
- **Employee** – Employee records  
- **Skill** – Technical skills mapping  
- **Project** – Project assignment  
- **ArchiveRecord** – Exit records  

---

### 🔄 Workflow Example

1. HR logs in  
2. Searches employee  
3. Applies filters / keywords  
4. Views results  
5. Assigns employee to project  
6. System updates employee status

## 📂 Project Structure

```bash
TeamSync/
│
├── TeamSync Standalone.html      # Main UI entry file
│
├── decoded/                     # Extracted JavaScript assets
│   ├── *.js
│
├── decoded_all/                 # Complete decoded bundle files
│   ├── *.js
│   ├── *.txt
│
├── decoded_check/               # Verified & cleaned scripts
│   ├── *.js
│
├── tools/                       # Utility & helper scripts
│   ├── extract-bundler-asset.js
│   ├── replace-bundler-asset.js
│   ├── scan-bundler-manifest.js
│   ├── dump-all-js-assets.js
│   └── sidebar.js
```

---

### 📌 Folder Description

* **TeamSync Standalone.html** → Main UI file that runs the application
* **decoded/** → Contains extracted JavaScript from bundled files
* **decoded_all/** → Full raw decoded output for analysis/debugging
* **decoded_check/** → Cleaned and verified scripts ready for use
* **tools/** → Custom scripts used for extraction, replacement, and debugging
